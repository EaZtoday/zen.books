Step 2: Create SchemaRenderer Component
  // Create: components/SchemaRenderer.tsx

  export function SchemaRenderer({ schemas }) {
    return (
      <>
        {schemas.map((schema, i) => (
          <script
            key={i}
            type="application/ld+json"
            dangerouslySetInnerHTML={{ __html: JSON.stringify(schema) }}
          />
        ))}
      </>
    );
  }

Step 3: Use in Blog Post (example for interview guide)
  import SchemaRenderer from '@/components/SchemaRenderer';
  import interviewGuideSchema from '@/schema/interview-guide-schema.json';

COMMON IMPLEMENTATION MISTAKES TO AVOID
========================================

❌ WRONG: Putting schema in robots.txt or sitemap
✓ RIGHT: Inject schema into <head> or <body> of HTML page

❌ WRONG: Using <script type="text/plain">
✓ RIGHT: Using <script type="application/ld+json">

❌ WRONG: Minifying/compressing the JSON-LD
✓ RIGHT: Keep JSON readable (affects debugging)

❌ WRONG: Hardcoding URLs instead of using canonical
✓ RIGHT: Use consistent domain across all schema

❌ WRONG: Creating single schema for all posts
✓ RIGHT: Unique schema per article (unique headline, datePublished)

❌ WRONG: Ignoring dateModified field
✓ RIGHT: Update dateModified when article changes
